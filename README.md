# 📝 gRPC Todo List (Go)

Простой и эффективный сервис для управления списком задач, написанный на **Go**. Проект объединяет строго типизированное gRPC API, REST-шлюз и интерактивный веб-интерфейс на **HTMX**. В качестве хранилища используется **Redis** с настроенной политикой очистки данных.

---

## 🏗️ Архитектура

Проект разделен на несколько функциональных слоев:

1.  **gRPC Server (порт 50051)**: Основной интерфейс для выполнения CRUD-операций. Использует Protobuf для описания контрактов и генерации кода.
2.  **HTTP Gateway (порт 80)**: Прокси на базе `grpc-gateway`. Транслирует REST-запросы в gRPC-вызовы, предоставляя доступ к API обычным HTTP-клиентам. Включает встроенную Swagger-документацию.
3.  **HTMX Web UI**: Веб-интерфейс на стандартных Go-шаблонах. Благодаря HTMX реализует динамическое обновление страниц без использования тяжелых JS-фреймворков.
4.  **Redis Storage**: Хранит данные задач и индексы пользователей.

---

## 🔑 Идентификация пользователей

Сервис поддерживает два способа передачи идентификатора пользователя (`user_id`), что удобно для разных типов клиентов:

| Способ | Параметр | Применение |
| :--- | :--- | :--- |
| **Header** | `X-User-ID` | Интеграция через API (gRPC / REST) |
| **Cookie** | `X-User-ID` | Работа через браузер (Web UI) |

---

## ⏱️ Особенности хранения (TTL)

Для оптимизации ресурсов в демонстрационном режиме используется механизм автоматического удаления данных. Каждая запись в Redis имеет **срок жизни 15 минут**. Время хранения обновляется при любом изменении задачи (создание или редактирование).

## 🚦 Запуск и развертывание

### 1. Через Docker (быстрый старт)
Проект настроен на работу во внешней сети `projects-network`.

```bash
# Создание сети (если нет)
docker network create projects-network

# Сборка и запуск
docker-compose up -d --build
```

### 2. Локальный запуск
Для разработки и тестирования:

```bash
# Запуск Redis
docker run -d --name redis -p 6379:6379 redis

# Генерация кода из .proto
make gen

# Запуск приложения
go run cmd/app/main.go
```

**Доступные эндпоинты:**
- **Web UI**: `http://localhost/`
- **Swagger**: `http://localhost/swagger`
- **REST API**: `http://localhost/api/v1/todos`
- **gRPC API**: `localhost:50051`
